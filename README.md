# High Point Market Exhibitor Scraper

Scrapes the [High Point Market Exhibitor Directory](https://www.highpointmarket.org/ExhibitorDirectory) and outputs a formatted Excel file with one row per exhibitor.

**No login required** — the directory is publicly accessible.

---

## What It Scrapes

| Field | Source |
|---|---|
| Company Name | `h1` inside `.exhibitor-contain` |
| Exhibitor ID | numeric ID from the URL path |
| Location / Booth | e.g. `IHFC - C206, Commerce, Floor 2` |
| Shuttle Stop | labeled span in `.info-block` |
| Neighborhood | labeled span in `.info-block` |
| Phone | `Corporate Phone:` span |
| Website | external link in `.info-block` |
| Instagram | `.info-block ul.social a.inst` |
| Facebook | `.info-block ul.social a.fb` |
| YouTube | `.info-block ul.social a.yt` |
| Pinterest | `.info-block ul.social a.pint` |
| Twitter | `.info-block ul.social a.twt` |
| LinkedIn | `.info-block ul.social a.li` |
| Description | "Who We Are" tab content |
| Video URLs | YouTube iframe `src` values (pipe-separated) |
| Gallery Image URLs | `/images/exhibitor/gallery/` img srcs (pipe-separated) |
| Source URL | original input URL |
| Status | `ok` / `timeout` / `error: ...` |

## Local Setup

### Prerequisites
- Python 3.9+
- pip

### Install

```bash
pip install -r requirements.txt
playwright install chromium
```

---

## Step 1 — Harvest URLs

Run once to build `urls.xlsx` containing all exhibitor page URLs.

```bash
python harvest_urls.py
```

This opens a headless browser, iterates A–Z alpha filters on the exhibitor directory, and saves every `/exhibitor/<id>` URL to `urls.xlsx`.

**After it completes:** commit `urls.xlsx` to your repository so GitHub Actions can use it.

```bash
git add urls.xlsx
git commit -m "Add exhibitor URLs"
git push
```

---

## Step 2 — Scrape Exhibitor Pages

With `urls.xlsx` in place:

```bash
# Test run — scrape only first 3 URLs
TEST_MODE=3 python scraper.py

# Full run
python scraper.py
```

Output: `exhibitors_output.xlsx`

### Resuming an interrupted run

If the scraper is interrupted, progress is saved to `progress.json`. Re-running `python scraper.py` will skip already-completed URLs automatically.

To start fresh (ignore progress):
```bash
rm progress.json
python scraper.py
```

## GitHub Actions

### Workflow: `scrape.yml`

Go to **Actions → High Point Market Scraper → Run workflow** and choose a mode:

| Mode | What it does |
|---|---|
| `harvest_urls` | Crawls directory → uploads `urls.xlsx` as artifact |
| `scrape_only` | Scrapes exhibitor pages using `urls.xlsx` in the repo |
| `harvest_then_scrape` | Both steps in sequence |

**Test mode dropdown:** defaults to `yes` (only 3 URLs). Set to `no` for a full run.

### Workflow for first-time setup

1. Run `harvest_urls` mode → download the `exhibitor-urls` artifact → commit `urls.xlsx` to the repo
2. Run `scrape_only` mode in test mode first to verify selectors work
3. Run `scrape_only` with test mode `no` for the full run
4. Download the `exhibitor-results` artifact to get `exhibitors_output.xlsx`

## Troubleshooting

| Symptom | Fix |
|---|---|
| `urls.xlsx not found` | Run `harvest_urls.py` first, or commit the file |
| Empty fields | Open the page manually and inspect — selectors may have changed |
| All rows are timeouts | The site may be rate-limiting; reduce `CONCURRENCY` in `scraper.py` |
| `libasound2` error on Actions | Workflow already pins `ubuntu-22.04` — don't change to `ubuntu-latest` |
| Harvest finds 0 URLs | The directory JS may have changed; run with `headless=False` locally to debug |
